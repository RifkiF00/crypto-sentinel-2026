00 — Overview

Apa Itu Crypto-Sentinel?

Crypto-Sentinel bukan pengganti Core Banking. Ia adalah lapisan penjaga yang duduk di antara aplikasi nasabah dan database bank — memotong setiap transaksi mencurigakan sebelum dana keluar dari ekosistem perbankan Indonesia.

The Brain
Graph Neural Network
Arsitektur ST-GCN (Spatio-Temporal GCN) yang memetakan jaringan rekening seperti "jaring laba-laba". Mendeteksi pola smurfing, layering, dan fraud ring dalam hitungan milidetik menggunakan analitik relasional.
The Enforcer
Smart Circuit Breaker
Mekanisme interupsi otomatis berbasis API SNAP BI. Jika Risk Score >85, transaksi ditahan sebelum dana keluar. Latensi target <50ms — lebih cepat dari kedipan mata manusia.
The Interface
Forensic Dashboard
Antarmuka Explainable AI berbasis SHAP yang menampilkan visualisasi graf jaringan fraud secara real-time. Auto-generate draf Suspicious Transaction Report (STR) untuk PPATK/OJK.
Velocity Gap — Masalah Inti

BI-FAST menyelesaikan transaksi dalam T+0 (detik), namun FDS konvensional masih bekerja secara reaktif dalam T+1 (berkala). Sindikat kejahatan mengeksploitasi celah ini untuk memecah dana ke jaringan mule account dan mengonversinya ke kripto internasional sebelum sistem sempat bereaksi — kerugian tercatat Rp800 miliar di delapan bank (PPATK, 2024).

Risk Score Kalkulasi Formula
// Risk Score Aggregation — Weighted Sum (0–100)

RiskScore = (
  // Group 1: Behavioral (bobot 20%)
  W_velocity   × score_velocity       +   // 0.08
  W_odd_hour    × score_odd_hour       +   // 0.05
  W_dormant     × score_dormant        +   // 0.04
  W_profile     × score_profile_anomaly+   // 0.03

  // Group 2: Graph Intelligence (bobot 45% — paling kritis)
  W_mule_ring   × score_mule_ring      +   // 0.25
  W_layering    × score_layering       +   // 0.12
  W_blacklist   × score_blacklist_hit  +   // 0.08

  // Group 3: Purpose Mismatch (bobot 20%)
  W_purpose     × score_purpose_mismatch+  // 0.15
  W_ledger      × score_ledger_mismatch+   // 0.05

  // Group 4: Technical (bobot 15%)
  W_travel      × score_impossible_travel+  // 0.10
  W_geo         × score_geolocation    +   // 0.05
  W_device      × score_device_integrity    // 0.00 (roadmap)
)

// DECISION LOGIC
if (RiskScore > 85)  → "BLOCK"   // Circuit Breaker aktif
if (RiskScore > 60)  → "REVIEW"  // Flag untuk analis
if (RiskScore ≤ 60)  → "ALLOW"   // Teruskan ke core banking

# 1. BANGUN GRAF DARI DATA TRANSAKSI
G = nx.DiGraph()

# Tambah node (setiap rekening unik)
for account in accounts:

        account.balance,           # Saldo
        account.transaction_count, # Frekuensi
        account.age_days,          # Usia rekening

for tx in transactions:

08 — Demo Script

Skenario Demo di Depan Juri

Tiga skenario yang harus bisa didemonstrasikan secara live. Urutan ini dirancang untuk membangun ketegangan naratif dan mencapai "wow moment" maksimal.

Skenario 1 — Normal
Transaksi Aman ✓
Billy transfer Rp500.000 ke rekening teman di siang hari dari IP Jakarta. Sentinel menganalisis → Risk Score 12 → ALLOW → Dana terkirim → Dashboard menampilkan status hijau.

Tujuan: Tunjukkan sistem berjalan normal untuk transaksi biasa.
Skenario 2 — Suspicious
Transaksi Mencurigakan ⚠
Rifki transfer Rp250 juta, jam 02:30 pagi, tujuan "Crypto Exchange", berita "Bayar Hutang". Sentinel menganalisis → Risk Score 94 → BLOCKED → Dashboard alert merah → STR draft otomatis terbentuk.

Tujuan: Tunjukkan Circuit Breaker bekerja real-time.
Skenario 3 — Attack
Serangan Smurfing 🎯
Billy aktifkan Attack Simulator → 100 transaksi kecil dari berbagai akun ke satu mule account dalam 20 detik → Dashboard menampilkan 98 BLOCKED secara live → GNN graf menampilkan "jaring laba-laba" terbentuk visual → Rp490 juta diselamatkan.

Tujuan: KLIMAKS — momen yang tidak terlupakan juri.
💡 Tips Demo Kritis untuk Aam (Frontend)

Dashboard adalah senjata visual utama tim EXPRESSO. Juri akan menghabiskan waktu paling lama melihat layar ini. Pastikan: (1) GNN Graf bergerak dan berubah saat transaksi masuk, (2) Risk Score gauge beranimasi, (3) Alert feed real-time terlihat jelas, (4) Angka "Dana Diselamatkan" ter-update live. Investasikan 40% waktu coding di dashboard ini — return-nya paling tinggi.

10 — Deployment

Cara Menjalankan Sistem

docker-compose.yml — Satu Perintah untuk Semua Service
# docker-compose.yml
# Jalankan: docker-compose up --build

version: '3.8'

services:

  # 1. DATABASE
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: sentinel_db
      POSTGRES_USER: sentinel_user
      POSTGRES_PASSWORD: sentinel_pass
    ports: ["5432:5432"]
    volumes: [postgres_data:/var/lib/postgresql/data]

  # 2. REDIS CACHE (Blacklist)
  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  # 3. SENTINEL AI ENGINE (Rifki)
  sentinel-ai:
    build: ./sentinel-ai
    ports: ["8001:8001"]
    environment:
      DB_URL: postgresql://sentinel_user:sentinel_pass@postgres/sentinel_db
      REDIS_URL: redis://redis:6379
    depends_on: [postgres, redis]

  # 4. CORE BANKING MOCK (Desta)
  core-banking:
    build: ./core-banking
    ports: ["8002:8002"]
    environment:
      DB_URL: postgresql://sentinel_user:sentinel_pass@postgres/sentinel_db
      SENTINEL_URL: http://sentinel-ai:8001
    depends_on: [sentinel-ai]

  # 5. DASHBOARD FRONTEND (Aam)
  dashboard:
    build: ./dashboard-frontend
    ports: ["3000:3000"]
    environment:
      REACT_APP_API_URL: http://core-banking:8002
      REACT_APP_WS_URL: ws://core-banking:8002/ws
    depends_on: [core-banking]

volumes:
  postgres_data:

---

## Mobile Banking App  →  http://localhost:3000/app
## Forensic Dashboard  →  http://localhost:3000/dashboard
## Sentinel API Docs   →  http://localhost:8001/docs
## Core Banking Docs   →  http://localhost:8002/docs
